# Embedding/Head 变体预训练实验报告

## 1. 实验概述

本报告记录 MiniMind dense 预训练阶段的 embedding/head 参数化对照实验。实验只覆盖 pretrain，不包含 SFT、RL、DPO 或其他后训练流程。

实验分为两组：

- MiniMind 原始预训练数据：完整训练 S1-S12，共 12 个变体，每个变体运行 `seed=42,123,2026` 三个随机种子。
- FineWeb-Edu/GPT-2 数据：迁移到 FineWeb-Edu packed 6B token 数据和 GPT-2 tokenizer，训练关键变体 `S1,S2,S3,S4,S6,S7,S11,S12`，同样运行三个随机种子。

核心问题是比较 tied/untied 架构、输入侧修改、输出侧修改、低秩改造以及组合变体对验证集 loss 和 PPL 的影响。

## 2. 实验变体

所有变体均基于 MiniMind dense 模型，hidden size 为 `768`。低秩相关变体统一使用 `rank=32`，并开启 `lm_head_bias=True`。

| 变体 | tied | 说明 |
| --- | --- | --- |
| S1 | 是 | 标准 tied baseline，输入 embedding 与 `lm_head.weight` 共享 |
| S2 | 否 | 标准 untied baseline，输入 embedding 与输出 head 独立 |
| S3 | 是 | 输入侧 hidden 维全局平移，形式为 `embedding - beta` |
| S4 | 是 | 输入侧词表维加性 LoRA |
| S5 | 是 | 输出侧词表维加性 LoRA |
| S6 | 是 | 输入侧乘性低秩变换 |
| S7 | 是 | 输出侧乘性低秩变换 |
| S8 | 否 | untied + 输入侧 hidden 维全局平移 |
| S9 | 否 | untied + 输入侧词表维加性 LoRA |
| S10 | 否 | untied + 输入侧乘性低秩变换 |
| S11 | 是 | 输出侧 hidden 维全局平移 |
| S12 | 是 | S3 + S6，输入侧 hidden 维全局平移与输入侧乘性低秩组合 |

S12 的输入 embedding 有效形式为：

```text
embedding_eff = embedding - embedding_beta + embedding_mul_b(embedding_mul_a(embedding))
```

S11 的输出侧 `beta` 是 hidden 维参数，不是 `lm_head` bias。`lm_head_bias` 在所有新版实验中单独开启。

### 2.1 LoRA/低秩形式与参数增量

记词表大小为 `V`，hidden size 为 `H=768`，低秩 rank 为 `r=32`。下面的参数增量均相对于同一 tokenizer 下的 S1 baseline 计算，且包含本实验统一开启的 `lm_head_bias`。

S1 参数量：

- MiniMind tokenizer：`63,918,592`
- GPT-2 tokenizer：`97,644,625`

各类新增参数形式：

- hidden 维平移：`beta in R^H`，参数增量为 `H=768`。
- 输入侧词表维加性 LoRA：`delta_e = B(A[token_id])`，其中 `A in R^{V x r}`，`B in R^{r x H}`，参数增量为 `V*r + H*r`。
- 输出侧词表维加性 LoRA：`logits = h W^T + (B_o(h)) A_o^T`，其中 `A_o in R^{V x r}`，`B_o in R^{H x r}`，参数增量为 `V*r + H*r`。
- 输入侧乘性低秩变换：`e_eff = e + B_m(A_m(e))`，其中 `A_m in R^{H x r}`，`B_m in R^{r x H}`，参数增量为 `2*H*r`。
- 输出侧乘性低秩变换：`logits = lm_head(h + B_m(A_m(h)))`，参数增量同样为 `2*H*r`。
- S12：`e_eff = e - beta + B_m(A_m(e))`，参数增量为 `H + 2*H*r`。

相对于 S1 的参数增量比例：

| 变体/模块 | 新增参数公式 | MiniMind 参数增量 | MiniMind 增量比例 | GPT-2 参数增量 | GPT-2 增量比例 |
| --- | ---: | ---: | ---: | ---: | ---: |
| S3/S11 hidden beta | `H` | 768 | 0.0012% | 768 | 0.0008% |
| S4/S5 词表维加性 LoRA | `V*r + H*r` | 229,376 | 0.3589% | 1,632,800 | 1.6722% |
| S6/S7 hidden 乘性 LoRA | `2*H*r` | 49,152 | 0.0769% | 49,152 | 0.0503% |
| S12 beta + hidden 乘性 LoRA | `H + 2*H*r` | 49,920 | 0.0781% | 49,920 | 0.0511% |
| S2 untied head 权重 | `V*H` | 4,915,200 | 7.6898% | 38,597,376 | 39.5284% |
| S9 untied + 输入侧词表维 LoRA | `V*H + V*r + H*r` | 5,144,576 | 8.0486% | 40,230,176 | 41.2006% |
| S10 untied + hidden 乘性 LoRA | `V*H + 2*H*r` | 4,964,352 | 7.7667% | 38,646,528 | 39.5788% |

可以看到，词表维 LoRA 的成本随 `V` 线性增长，因此在 GPT-2 词表下 S4/S5 的参数增量明显高于 MiniMind tokenizer。hidden 维乘性 LoRA 只依赖 `H` 和 `r`，因此绝对参数增量在两组实验中相同。S12 在参数成本上接近 S6/S7，但实验结果明显更强，说明其收益主要来自输入侧平移与乘性低秩组合，而不是显著增加参数量。

## 6. 跨数据集结论

输入侧修改是最稳定的收益来源。MiniMind 上 S3 最优、S12 第二；FineEdu/GPT-2 上 S12 最优、S3 第二。两组实验都支持输入侧 hidden 平移和输入侧低秩组合是值得继续研究的方向。

S12 在更通用的 FineEdu/GPT-2 设置下表现最好，说明 `S3 + S6` 的近似仿射形式比单独 S3 更有跨数据/跨 tokenizer 的泛化潜力。

输出侧单独改造收益不稳定。S11 在 FineEdu 上略优于 S1，但在 MiniMind 上基本持平；S7 在两个数据集上都没有稳定超过 S1。

纯 untied baseline S2 在两组实验中都明显弱于 tied baseline S1。MiniMind 上 S2 比 S1 高 `0.062159`，FineEdu/GPT-2 上高 `0.074002`。这说明在当前模型规模、训练 token 数和训练配置下，直接 untie embedding/head 不是有效方向。

## 7. 后续建议

优先继续围绕 S3 和 S12 做扩展实验。S3 可作为简单强 baseline；S12 可作为更有泛化潜力的组合方案。

如果继续做结构改造，建议优先检查：

- S12 中 `embedding_beta` 与乘性 LoRA 的相对梯度强度。
- S3/S12 的 embedding 与 unembedding 梯度路径差异。
- FineEdu/GPT-2 上 S12 是否在更长训练 token 或更多 epoch 下继续扩大优势。
- `rank=16/32/64` 对 S4/S6/S12 的影响。

梯度分析可直接从 FineEdu 日志目录中的 `*_grad_stats.jsonl` 和 `*_grad_tensors/` 读取。

### 8.3 补训结论

S13 的结论是数据集相关的：MiniMind 上不稳定，平均略差于 S1；FineEdu/GPT-2 上稳定优于 S1，并超过 S3 成为第二名，但仍落后于 S12。说明 `S4 + S6` 的组合在更大、更通用的数据上有收益，但不如 `S3 + S6` 稳定。

S5 在 FineEdu/GPT-2 上平均略优于 S1，但收益很小，且 seed2026 差于 S1。它可以作为输出侧改造的补充对照，但不是当前最值得继续扩展的方向。

综合两组数据，当前优先级仍应是 S12、S3，其次观察 FineEdu 上的 S13。MiniMind 上的 S13 不建议作为主线方案继续扩大训练，除非后续专门研究两条输入侧 LoRA 分支的梯度竞争或正交约束。

### 8.4 FineEdu/GPT-2 上 S13/S5 梯度关系

本轮 S13/S5 补训中，FineEdu/GPT-2 保存了 `*_grad_stats.jsonl`；MiniMind 补训当时使用 `GRAD_LOG_INTERVAL=0`，没有保存模块梯度统计。因此这里的梯度关系分析只覆盖 FineEdu/GPT-2。汇总文件：

- `logs/fineedu_s13_s5_grad_relation_summary.csv`
- `logs/fineedu_s13_s5_grad_relation_summary.md`

S13 的两个输入侧分支梯度强度如下：

| seed | vocab LoRA mean norm | mul LoRA mean norm | mul / vocab | vocab / shared | mul / shared | embed-unemb cosine | unemb / embed |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 42 | 0.011519 | 0.019523 | 1.69x | 14.67% | 24.86% | -0.0020 | 1.90x |
| 123 | 0.011737 | 0.013963 | 1.19x | 15.17% | 18.05% | -0.0014 | 1.94x |
| 2026 | 0.011942 | 0.015246 | 1.28x | 15.29% | 19.52% | -0.0005 | 1.91x |

S13 中 hidden 乘性 LoRA 分支的梯度范数整体大于词表加性 LoRA 分支，尤其 seed42 达到 `1.69x`。但更好的 seed123 反而是两个分支最均衡的一组，`mul/vocab=1.19x`，对应 loss 相对 S1 改善 `-0.008652`。seed2026 改善最小，loss 只比 S1 低 `0.000524`，其 `mul/vocab=1.28x`，不是最大失衡，但收益明显变弱。

S5 的输出侧 LoRA 梯度强度如下：

| seed | output LoRA mean norm | output / shared | output_lora_a | output_lora_b | embed-unemb cosine | unemb / embed |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| 42 | 0.023013 | 29.31% | 0.018931 | 0.012506 | -0.0016 | 1.93x |
| 123 | 0.027557 | 34.92% | 0.024475 | 0.012263 | -0.0003 | 1.90x |
| 2026 | 0.023584 | 30.33% | 0.019766 | 0.012605 | -0.0006 | 1.95x |

S5 的输出 LoRA 梯度占 shared embedding/head 梯度约 `29%~35%`，并不弱。seed123 的输出 LoRA 梯度最强，对应效果也最好；seed2026 的输出 LoRA 梯度强度接近 seed42，但效果差于 S1，说明 S5 的问题不是简单的“输出 LoRA 没学到”，更可能是输出侧修正与 tied shared weight 的主梯度方向不稳定，部分 seed 会带来轻微负收益。

各变体的 tied split 关系整体很接近：`unemb_grad / embed_grad` 基本都在 `1.85x~1.98x`，`embed_unemb_grad_cosine` 接近 0。也就是说，S13/S5 没有表现出明显的 embedding 与 unembedding 梯度方向冲突；当前更值得关注的是 LoRA 分支本身的相对强度和是否抢占 shared weight 的有效更新。

对 S13 来说，FineEdu 上的证据支持一个弱判断：`S4 + S6` 能带来收益，但两个输入侧分支并不是简单相加越强越好，较均衡的分支比例可能更稳。对 S5 来说，输出侧 LoRA 梯度足够大，但收益小且 seed2026 失效，说明输出侧单独修正的泛化稳定性仍弱于输入侧方案。
