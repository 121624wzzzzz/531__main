# S1-S12 Embedding/Head 变体预训练实验记录

## 实验目标

本实验比较 MiniMind dense 模型在不同 embedding/head 参数化方式下的预训练验证集 loss。实验只做 pretrain 阶段，不做 SFT、RL、LoRA 微调等后续阶段。

核心对照包括标准 tied、标准 untied、输入侧/输出侧加性 LoRA、乘性低秩变换、hidden 维全局平移，以及组合变体 S12。

## 数据与 Tokenizer

- 训练/验证数据：`dataset/pretrain_t2t.jsonl`
- 训练切分：前 99%，由 `--train_split_ratio 0.99` 控制
- 验证切分：同一数据文件尾部 1%
- 验证起点：`start_index=8384138`
- 验证样本数：`84689`
- 验证 token 数：`27069543`
- Tokenizer：本地 MiniMind tokenizer，路径为 `model`
- 最大序列长度：`max_seq_len=340`

训练和验证使用同一 tokenizer。验证脚本中通过 `AutoTokenizer.from_pretrained("model")` 加载 tokenizer，并对样本添加 BOS/EOS，padding token 的 label 置为 `-100`。

- 模型：MiniMind dense，非 MoE
- hidden size：`768`
- hidden layers：`8`
- low-rank rank：`32`
- `lm_head_bias`：开启，即 `--lm_head_bias 1`

- batch size：`224`
- accumulation steps：`1`

- epochs：`2`
- learning rate：`5e-4`
- dtype：`bfloat16`
- 随机种子：`42`、`123`、`2026`

## 变体说明

- S1：标准 tied，输入 embedding 和 `lm_head` 权重共享。
- S2：标准 untied，输入 embedding 和输出 head 独立。
- S3：tied，输入侧 hidden 维全局平移 `embedding - beta`。
- S4：tied，输入侧词表维加性 LoRA。
- S5：tied，输出侧词表维加性 LoRA。
- S6：tied，输入侧乘性低秩变换。
- S7：tied，输出侧乘性低秩变换。
- S8：untied，输入侧 hidden 维全局平移。
- S9：untied，输入侧词表维加性 LoRA。
- S10：untied，输入侧乘性低秩变换。
- S11：tied，输出侧 hidden 维全局平移。
- S12：tied，输入侧 hidden 维全局平移 + 输入侧乘性低秩变换，即近似仿射形式。

S12 的实现形式为：

```text
embedding_eff = embedding - embedding_beta + embedding_mul_b(embedding_mul_a(embedding))
```

## 主要结论

S3 是三 seed 平均验证 loss 最低的方案，平均比 S1 低 `0.011671`。S12 排名第二，平均比 S1 低 `0.008113`，说明输入侧全局平移与乘性低秩组合是有效的，但没有超过单独的 S3。

S4 和 S6 平均略优于 S1，但优势较小。S11 与 S1 基本持平。untied 系列整体明显弱于 tied 系列。
